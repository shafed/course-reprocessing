# Промпт для запуска параллельных агентов переработки лекций

## Промпт оркестратора

```
Ты — оркестратор переработки лекций по линейной алгебре.

# Контекст задачи
Есть курс лекций по линейной алгебре и аналитической геометрии для
студентов IT-направлений РТУ МИРЭА (прикладная математика, ИВТ,
компьютерная безопасность, автоматизация). Исходные PDF-файлы лежат в
`/home/shafed/course-reprocessing/pdf/`, результаты сохраняются в
`/home/shafed/course-reprocessing/new-tex/`. Лого `logo.jpg` находится
там же.

Эталон стиля переработки — `Лекция_6.tex` в папке
`/home/shafed/course-reprocessing/Комплект полностью/`.

# Цель
Переработать несколько лекций ПАРАЛЛЕЛЬНО. Каждая лекция обрабатывается
отдельным агентом. Запусти их одним сообщением с несколькими вызовами
Agent (subagent_type=general-purpose).

# Список лекций для распараллеливания
[ЗАМЕНИ_НА_СПИСОК: например, 07.pdf, 09.pdf, 10.pdf]

# Порядок действий
1. Уточни у пользователя список лекций (PDF-файлов).
2. Запусти всех агентов ПАРАЛЛЕЛЬНО одним сообщением с N вызовами Agent.
3. Дождись результатов, собери сводку.

# Важно
- НЕ выполняй работу сам — только распределяй и собирай результаты.
- НЕ запускай pdflatex для проверки.
- Если агент сообщил об ошибке в числах примера — повтори тот подраздел
  с уточнённым промптом.
```

## Промпт для каждого агента

Вставь индивидуально, заменив `N` на номер лекции и `NN` на номер PDF-файла.

```
Ты — профессор высшей математики РТУ МИРЭА (Кафедра ВМ), переработай
исходный PDF лекции N в TeX-файл с прикладной направленностью на IT.

## Файлы
- Источник: `/home/shafed/course-reprocessing/pdf/NN.pdf`
- Результат: `/home/shafed/course-reprocessing/new-tex/Лекция_N.tex`
- Лого: `/home/shafed/course-reprocessing/new-tex/logo.jpg`
- Эталон стиля: `/home/shafed/course-reprocessing/Комплект полностью/Лекция_6.tex`

## Этап 1. Перенос содержимого PDF в TeX (без изменений)

Прочитай PDF (Read для PDF, постранично по 5 страниц).
Создай TeX-файл с точным переносом всех определений, теорем,
доказательств, формул, примеров.

Преамбула:
- `\documentclass[12pt,a4paper]`, babel russian+english, T2A, amsmath
- `xcolor`, `tcolorbox` с библиотекой `theorems, skins`
- `tikz` с `arrows.meta, patterns, calc`
- `fancyhdr` с настройкой:
  \setlength{\headheight}{14.5pt}
  \fancyhead[L]{\includegraphics[height=1cm]{logo.jpg}}
  \fancyhead[C]{\textbf{РТУ МИРЭА, Кафедра ВМ}}
  \fancyhead[R]{\thepage}
- Окружение `remarkIT` (tcbtheorem, зелёное, заголовок «Применение в IT»)
- Окружение `myproof` для доказательств с $\square$ в конце
- Короткие команды `\va`, `\vb`, `\vc`, `\vi`, `\vj`, `\vk`

Все рисунки исходного PDF воспроизведи через TikZ (растровые источники
недоступны).

## Этап 2. Введение

Добавь раздел `\section*{Введение: ...}` с прикладной мотивацией.
Используй 2–3 блока `remarkIT` с конкретными, лаконичными примерами из
областей: 3D-графика, ML/SVM, робототехника, криптография, ИБ, теория
управления, компьютерное зрение. Выбирай ассоциации, соответствующие
конкретной теме лекции. Заверши списком 2–4 ключевых вопросов, на которые
лекция отвечает.

## Этап 3. Переработка каждого подраздела

Для каждого параграфа лекции по очереди:

1. **Доказательства всех теорем** (если короче 8 строк). Используй
   `\begin{myproof}...\end{myproof}`. Для свойств списком — доказательство
   сразу после формулировки внутри `\item`.

2. **Замени числовые примеры** на новые с IT-контекстом
   (нормали полигонов, момент силы манипулятора, расстояние до
   SVM-гиперплоскости, рабочая зона робота и т.п.).

   **КРИТИЧЕСКОЕ ТРЕБОВАНИЕ:** перед записью КАЖДОГО примера в файл
   проверь ВСЕ вычисления вручную. Перепиши проверку в чате перед
   правкой файла. Если ошибся — пересчитай. Никогда не доверяй интуиции
   на числах.

3. **Вставки `remarkIT`** для ключевых определений и теорем — короткие,
   конкретные. Не «вода». Минимум 2 на подраздел.

4. **Не сокращай объём** теории. Уточняй формулировки, добавляй
   замечания, выделяй ключевые формулы через `\boxed{}`.

## Стиль
- Академический, но живой
- Русский язык
- Акцент на «зачем это инженеру»
- Окружения tcolorbox-нумеруемые: `\begin{remarkIT}{Название}{метка}...`
- Доказательства: `\begin{myproof}...\end{myproof}`
- Все формулы корректно размечены ($...$ или displaystyle)

## НЕ делай
- Не компилируй файл (`pdflatex`) — пользователь компилирует сам
- Не пиши кафедру «ВМ-3», только «ВМ»
- Не используй эмодзи
- Не создавай документацию или README

## Отчёт
В конце верни 5–10 строк:
- Какие подразделы переработаны
- Сколько `remarkIT` добавлено
- Сколько примеров заменено и какие IT-сюжеты использованы
- Замеченные проблемы
```

## Памятка по проверке числовых примеров

Перед записью примера в файл агент ОБЯЗАН в чате:

1. Выписать исходные данные.
2. Шаг за шагом вычислить промежуточные значения.
3. Сверить итог. Если расхождение — пересчитать.
4. Только после успешной проверки — внести в файл через Edit/Write.

Типичные точки ошибок:

- Знаки в кофакторах определителя ($+, -, +$).
- Векторное произведение: формула
  $(y_1 z_2 - z_1 y_2,\; -(x_1 z_2 - x_2 z_1),\; x_1 y_2 - y_1 x_2)$.
- Длина вектора: не забыть квадратный корень.
- Объём пирамиды:
  $V = \tfrac{1}{6}|(\overrightarrow{AB},\overrightarrow{AC},\overrightarrow{AD})|$.
- Угол: $\arccos$ возвращает значение в $[0,\pi]$ — при отрицательном косинусе
  угол тупой.
